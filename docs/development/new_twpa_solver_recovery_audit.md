# New TWPA Solver Recovery Audit

The previous `twpa_solver` IPM topologies are reduced sandbox models. They are useful for solver development, but they are not the Harmonia old-IPM design used by the historical Julia map.

The corrected source of truth is the Julia circuit generated by:

```text
Harmonia.jl/experiments/jc_setup_cache/run_report_old_ipm_power_map_gridn.jl
build_old_ipm_circuit()
```

That builder defines `Nj = 418 * 6`, phase-matching insertions, ports `1/2/4`, the `32 dB` source-power offset, `Npumpharmonics = (10,)`, and `Nmodulationharmonics = (5,)`.

## Surrogate Code Status

`ipm_jtwpa_reduced_marker`, `ipm_jtwpa_physical_coupler`, and `ipm_jtwpa_old_constants_compact_surrogate` are sandbox topologies. The deprecated alias `ipm_jtwpa_old_julia_parity` remains for backward compatibility, but its metadata now warns that it is not old-Julia circuit parity.

Numerical parity must not be claimed from these topologies.

## Correct Recovery Architecture

```text
Harmonia Julia design/map frontend
  -> build_old_ipm_circuit()
  -> neutral JSON netlist export
  -> independent backend importer/solver
```

The independent backend now starts from:

```text
Harmonia.jl/experiments/jc_setup_cache/export_old_ipm_circuit_json.jl
twpa_solver/importers/julia_circuit_json.py
twpa_solver/experiments/run_exported_julia_circuit_map.py
```
